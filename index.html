Highlight #1 About Version Control

Version control is a system that records changes to a file or set of files over time so that you can reference those specific verisons later when needed. Keeping every version of a file is imparitive to almost every profession in technology.

Highlight #2 Short History of Git

Git is the Linux communities own tool for a speedy, simple designed, strong supported, fully distributed system. Created in 2005 is has evolved and matured to be easy to use and yet retain these inital qualities.

Highlight #3 Git Basics

Git is a version-control system for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for source-code management in software development, but it can be used to keep track of changes in any set of files.

Highlight #4 Recording Changes to the Repository

As you edit files, Git sees them as modified, because you've changed them since your last commit. As you work, you selectively stage these modified files and then commit all those staged changes, and the cycle repeats.

Highlight #5 GitHub

GitHub is the largest known host for Git repositories, and is the central point of collaboration for millions of developers and projects. A large percentage of all Git repositories are hosted on GitHub, and many opensource projects use it for Git hosting.
